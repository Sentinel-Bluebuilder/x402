How It Works

Request. Pay. Connect.

The x402 protocol handles everything. Agent sends one fetch call — payment, settlement, and VPN provisioning happen automatically.

1

Agent sends request

POST /vpn/connect/30days with a Sentinel address in the body. Server returns HTTP 402 Payment Required with a PAYMENT-REQUIRED header containing USDC amount, payTo address, and network (Base).

Agent → Server
2

Payment signed automatically

@x402/fetch reads the 402 response, signs an EIP-3009 transferWithAuthorization for the exact USDC amount, and resends the request with a PAYMENT-SIGNATURE header. The agent's EVM key never leaves the agent.

Base · USDC
3

Facilitator settles on-chain

Our self-hosted facilitator verifies the EIP-3009 signature and settles the USDC transfer on Base. Fully decentralized — no Coinbase, no third party. Payment confirmed in ~2 seconds.

Automatic
4

VPN provisioned on Sentinel

Server sends one atomic transaction: MsgShareSubscription + MsgGrantAllowance. Agent is added to our plan with a fee grant — zero gas on Sentinel. Agent can now start VPN sessions on any of 900+ nodes, handshaking directly with the node. We never see the tunnel or the traffic.

Agent ↔ Node

Security

Trust nothing. Verify everything.

The protocol is designed so that trust is unnecessary. Your keys, your tunnel, your traffic.

EVM key never leaves the agent

EIP-3009 transferWithAuthorization is signed locally. The facilitator only receives the signature, never the private key. Standard USDC, no approve needed.

Tunnel is agent-to-node

WireGuard/V2Ray handshake is direct between the agent and the VPN node. We never see the tunnel credentials, the encryption keys, or the traffic.

Self-hosted facilitator

No Coinbase. No third party. Our facilitator verifies EIP-3009 signatures and settles USDC on Base. Fully auditable on-chain. Open source.

Zero gas on Sentinel

Agent gets a fee grant — zero gas to start VPN sessions. The Sentinel key works with any operator, any plan, any node. No lock-in.
